## Задание 1. Рекурсия
### Текст задачи
Из заданного числа вычли сумму его цифр. Из результата вновь вычли сумму
его цифр и т.д. Через сколько таких действий получится нуль? 
### Алгоритм решения
1. Пользователь вводит целое число.
2. Предикат step рекурсивно вычисляет количество шагов, необходимых для сведения числа к нулю, где на каждом шаге из числа вычитается сумма его цифр.
3. Предикат sum_digit рекурсивно вычисляет сумму цифр числа.

## Задание 2. Списки
### Текст задачи
Выяснить, упорядочены ли элементы списка по возрастанию
### Алгоритм решения
1. Пользователь вводит количество элементов в списке.
2. Предикат read_list формирует список из введенных пользователем элементов с помощью рекурсии.
3. Далее предикат isSorted проверяет, является ли полученный список отсортированным по неубыванию (каждый следующий элемент больше или равен предыдущему).
4. Выводится исходный список и результат проверки.

## Задание 3. Обработка списков 
### Текст задачи
Определим множество как список без повторяющихся элементов. Найти
пересечение множеств. 
### Алгоритм решения
1. Пользователь вводит количество элеметов и сами элементы 2-х списоков.
2. Далее программа с помощью предиката listToSet из каждого списка удаляются повторяющиеся элементы. Это делается через предикат delete, который рекурсивно проходит список и удаляет все вхождения текущего элемента.
3. intersection находит пересечения множеств. Он перебирает элементы первого списка и для каждого проверяет с помощью isMember, содержится ли он во втором списке.
4. Вывод результата.

## Задание 4. Логическая задача
### Текст задачи
Задача «Валютные махинации».
В нарушении правил обмена валюты подозреваются четыре работника банка
— Антипов (А), Борисов (B), Цветков (С) и Дмитриев (D). Известно:
1) если А нарушил правила обмена валюты, то и В нарушил;
2) если В нарушил, то и С нарушил или А не нарушил;
3) если D не нарушил, то А нарушил, а С не нарушил;
4) если D нарушил, то и А нарушил.
Кто из подозреваемых нарушил правила обмена валюты?
### Алгоритм решения
1. Перебираем все возможные варианты нарушителей (A,B,C,D могут быть true или false).
2. Для каждой комбинации проверяем все 4 условия.
3. Выбор комбинации, где все 4 условия выполняются.
